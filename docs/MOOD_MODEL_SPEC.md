Задача: расширить «циферблат» (46 модальных пар Левашова) с одних только

Все аффекты ниже — **из данных ПТМ** (`music theory/data/*`), не выдуманы;
источник указан у каждого блока.

## 2. Почему не таблица «всех сочетаний»

Типов аккордов ~30–40, пар × 12 интервалов → **>19 000 комбинаций**. Их никто не
размечал, и это не нужно: ПТМ кладёт цвет в **триадный скелет**, а надстройки —
это «окраска сверху» (подтверждается данными в §3). Поэтому модель **слоёная**, а
не перечислительная.

## 4. Пространство аффектов (3 оси)

Малый целочисленный масштаб, ось ≈ −3…+3.

- **valence** — тьма ↔ свет
- **tension** — устойчиво ↔ напряжённо
- **color** — обычно ↔ колоритно/экзотично

Центроиды 5 категорий (проекция, выведена из §7; используется ТОЛЬКО для
математики модификаторов, базовая категория берётся из авторской таблицы):

| Категория | valence | tension | color |
|-----------|:---:|:---:|:---:|
| 1 Позитив | +2 | −1 | 0 |
| 2 Фантастика | 0 | +1 | +2 |
| 3 Сказочность | +1 | 0 | +2 |
| 4 Драматизм | −2 | +1 | 0 |
| 5 Опасность | −2 | +2 | +1 |

## 5. Правило вычисления `mood(A, B)`

```
1. skeleton = (triad(A), interval_semitones, triad(B))         # триада ∈ {maj,min,dim,aug}
2. base:
     если triad(A),triad(B) ∈ {maj,min}:  base = table46[skeleton]      # provenance: authored
     иначе (есть dim/aug):                 base = dim_aug_rule(skeleton) # provenance: heuristic
3. clock = geometry(A, B)                                        # без изменений
4. pos = centroid(base.category)                                # точка в (valence,tension,color)
5. pos += Σ modifier(color_content(A)) + Σ modifier(color_content(B))
          [+ voice_leading_nudge(A,B)]                           # §6, опционально
6. category_out = nearest_category(pos)
     если == base.category:  mood = base.mood + «цветовые пометки» надстроек   # provenance: authored+
     иначе:                  mood = compose(category_out, доминирующие цвет-слова) # provenance: derived
```

**Консервативность:** модификаторы в первую очередь **обогащают описание**
(«+ утончённость (Maj7)», «+ подвешенность (sus4)»), а категорию **флипают лишь
при явном сдвиге** (порог по расстоянию), чтобы не подрывать авторскую таблицу.

### `dim_aug_rule` (небольшой набор, не таблица)
- **dim / dim7 как участник пары:** базовая категория → 4 или 5 (по tension
  интервала: тритон/полутон → 5, мягче → 4). dim7 добавляет «портал» в описание.
- **aug как участник пары:** базовая категория → 2 (реже 3): «парящее»,
  фантастика/космос.
- Интервал и «время» считаются обычной геометрией (для aug якорь = корень; для
  dim якорь = корень, помечаем симметрию).

### 5.1 Ходы «на месте» (один корень, interval = 0) — обязательно

Кейс, который таблица 46 НЕ покрывает (кроме авторских `maj→min +0` и
`min→maj +0`): корень и терция те же, меняется только окраска — надстройки,
sus/add, dim/aug, альтерации. Примеры пользователя: `C→C11`, `C9→Csus2`,
`C7→C`, `C13→Cb5`, `C→C6`, `C→Cdim`.

Здесь настроение задаёт **не скелет, а ДЕЛЬТА окраски** `Δ = вектор(B) − вектор(A)`
(§4) + именованный архетип хода:

| Архетип | Условие (A → B) | Δ (valence/tension/color) | Описание |
|---------|-----------------|:---:|----------|
| **Разрешение** | сложнее → проще (7→△, sus→△, alt→△, dim→△) | +1 / −2 / 0 | снятие напряжения, покой, приход домой |
| **Уплотнение / напряжение** | проще → напряжённее (△→7, △→b5, △→alt, →dim, →aug) | −1 / +2 / +1 | нагнетание, подвешенность, ожидание события |
| **Раскрытие / эмбиент** | → sus2 / add9 / add11 / 11 | 0 / 0 / +2 | воздушно, мечтательно, просторно |
| **Потепление** | → 6 / maj7 / add9 / 9 | +1 / 0 / +1 | тепло, ностальгия, джазовая мягкость, красочность |
| **Затемнение** | → min / dim / b5 / b9 / b13 | −2 / +1 / +1 | тень, тревога, драматизация |
| **Парение** | → aug / #5 / #11 | 0 / +1 / +2 | «парящее», фантастика, экзотика |

Правило: `pos = centroid(триада A) ; pos += Δ ; category = nearest(pos)`;
описание берётся из архетипа + конкретных цвет-слов надстроек. Провенанс —
`derived` (для `maj→min +0` и `min→maj +0` остаётся `authored` из таблицы).

Проверочные ожидания (в тест §8): `C7→C` = разрешение (Позитив/покой);
`C→C11` / `C9→Csus2` = раскрытие; `C13→Cb5` = уплотнение/напряжение;
`C→C6` = потепление; `C→Cdim` = затемнение; `C→Caug` = парение.

## 6. Голосоведение (непрерывный нюанс, опционально)

Между **полными** аккордами A и B:
- общие тоны (пересечение pitch-class) → гладкость, снижает tension;
- суммарное минимальное полутоновое движение → трение;
- диссонанс (содержание тритонов/полутонов, интервал-вектор) → tension.

Даёт «интенсивность»/уверенность и лёгкий сдвиг оси tension; заодно — основа для
живой обратной связи «как звучит» в окне-циферблате.

## 7. Провенанс (честность оценки)

Каждый результат помечается:
- `authored` — обе триады maj/min, надстройки не флипнули категорию → **1:1 из
  таблицы 46**;
- `derived` — база из таблицы/правил + модификаторы сместили категорию;
- `heuristic` — dim/aug или экзотические сочетания без авторского якоря.

В UI показываем бейдж, как уже показываем «вне таблицы 46 пар».

## 8. Валидация (критерии приёмки Фазы 1)

1. **Регресс костяка:** для всех 46 авторских пар с чистыми трезвучиями модель
   без модификаторов обязана вернуть ту же категорию → 100 % совпадение
   (автотест по `modalChordPairs.json`).
2. **Спот-чек надстроек:** `Cmaj7→Dadd9` — остаётся Позитив, но «ярче/мечтательнее»;
   `Cdim→G` — напряжение (4–5); `C→B♭aug`/`C→Eaug` — «парящее»/фантастика (2);
   `Am→Fmaj7` — сказочность/тепло. Ожидаемые результаты фиксируем в тесте.
3. **Стабильность:** маленькое изменение аккорда (add9) не должно резко менять
   категорию — только описание/интенсивность.

## 11. Решения (утверждено)

1. Значения модификаторов §3.2 — **приняты как стартовые**.
2. Флип категории — **консервативный**: надстройки прежде всего меняют описание,
   категорию флипают только при явном сдвиге (порог по расстоянию).
3. Голосоведение (§6) — **отложено в Фазу 2**.
4. dim/aug — **вектор-модель без ручной таблицы**: сильные триадные вектора
   (§3.1) + геометрия интервала + вектор второго аккорда. Так честнее по теории
   (характер dim/aug устойчив к интервалу) и однородно с надстройками.

Костяк maj/min-пар (interval ≠ 0) по-прежнему берётся из **авторской таблицы 46**
как ground truth; вектор-модель — для dim/aug, ходов «на месте» (§5.1) и всего,
чего в таблице нет.
