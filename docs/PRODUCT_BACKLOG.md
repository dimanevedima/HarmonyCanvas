## P1.5 — вывод партий на разные инструменты

Важный факт: **Ableton Live сводит всю внутреннюю MIDI-маршрутизацию в один
канал** — выбрать отдельный MIDI-канал при роутинге с трека/плагина нельзя
(то же ограничение задокументировано у Scaler). Поэтому канальный роутинг
одного инстанса на разные инструменты в Ableton невозможен.

- ✅ каждая партия эмитится на своём канале (аккорды → Ch 1, голос N → Ch N+1) —
  это работает для прямого роутинга в Logic Pro 11 / Cubase / Reaper;
- ✅ панель «Выводит этот инстанс»: on/off для аккордов и каждого голоса;
  выбор запоминается по-инстансно (WebView-хранилище), учитывается в
  DAW-таймлайне и локальном превью;
- ✅ **мульти-инстанс для Ableton** (как у Scaler): несколько инстансов делят
  один эскиз через пикер «Эскиз», правки синхронны, каждый инстанс выводит свою
  партию на свой инструмент;
- осталось: (a) синхронизация живёт в WebView — инстанс-приёмник обновляет
  вывод, пока его окно открыто; для полной надёжности перенести опрос в
  процессор (C++); (b) превью одиночной ноты по клику идёт на Ch 1, не на канал
  активного голоса;
- альтернатива без мульти-инстанса — раздельный drag-out/экспорт партий в
  отдельные MIDI-клипы (P2).

## P2 — экспорт и drag-and-drop

- отдельный drag-out активного melody voice;
- drag-out всех melody voices;
- визуально различать drag handle и обычную кнопку;
- всегда оставлять альтернативную команду Export для пользователей без drag.

## UX-ограничения

- loop handles и chord/note resize не требуют пиксельной точности;
- drag начинается только после небольшого порога движения;
- критические операции имеют видимую кнопку и keyboard alternative;
- piano roll получает один предсказуемый вертикальный scroll region;
- playhead, loop и selection не должны конкурировать за один жест;
- интерактивные элементы имеют заметные focus/pressed/disabled states;
- цвет роли дополняется degree/role label и не является единственным сигналом.

## Не входит в этап отделения

Этап отделения считается чистой миграцией: существующие edit-команды и
визуальный результат должны остаться эквивалентными. Пункты выше реализуются
после автономного запуска без ReferenceCompare.
