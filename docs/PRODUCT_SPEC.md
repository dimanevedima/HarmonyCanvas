# Harmony Canvas — финальное техническое задание

Статус: утверждённый baseline для MVP
Целевая среда: Windows, Ableton Live 12 Suite, Harmony Canvas JUCE VST3
Рабочее название: Harmony Canvas

## 1. Цель

Создать MIDI-редактор внутри Ableton Live, в котором высоты подсвечиваются не
только по ладу, но и по функции относительно аккорда, звучащего в конкретный
момент времени.

Пользователь должен иметь возможность:

1. загрузить выбранный MIDI-клип Live в редактор одной кнопкой;
2. видеть аккорды и цветные гармонические роли на общей временной оси;
3. рисовать и редактировать MIDI-ноты;
4. слышать результат через инструмент Ableton;
5. обновить исходный клип, создать новый клип либо перетащить `.mid` мышью.

## 2. Не входит в задачу

- инъекция в GUI или перекраска нативного piano roll Live;
- замена Arrangement/Session View;
- обязательное распознавание гармонии из аудио;
- полноценная DAW внутри плагина;
- поддержка всех DAW в первом релизе;
- надёжный drag-in MIDI-клипа прямо с timeline Live.

## 3. Основной пользовательский сценарий

1. Пользователь открывает Ableton template `Harmony Canvas.als`.
2. Загружает Harmony Canvas VST3 на служебную дорожку.
3. Выбирает или открывает MIDI-клип в Live.
4. Нажимает `Pull Selected Clip`.
5. Выбирает гармонию из лаборатории либо вводит/редактирует её в chord lane.
6. Пишет мелодию в цветной сетке, слушая инструмент на receiver track.
7. Завершает работу одним из способов:
   - `Commit` — обновить связанный клип;
   - `Commit as New` — создать новую версию;
   - перетащить `MIDI` на нужную дорожку Live.

## 4. Интерфейс MVP

### 4.1 Верхняя панель

- статус соединения с Live;
- название связанного клипа;
- tonic и mode;
- размер: 4/4, 3/4, 6/8;
- snap: 1/4, 1/8, 1/16, 1/32;
- режим строк: Scale / Chromatic;
- цветовой режим: Roles / Degrees;
- `Pull`, `Commit`, `Commit as New`, draggable `MIDI`.

### 4.2 Piano roll

- вертикальный диапазон MIDI 0–127 с прокруткой;
- горизонтальная шкала в beats/bars;
- zoom и scroll;
- добавление ноты двойным кликом или Draw mode;
- выделение одиночное, множественное и рамкой;
- перемещение с привязкой к сетке;
- изменение длительности за правый край;
- delete, duplicate, copy/paste;
- velocity как минимум числом в inspector;
- playhead, loop region и audition выбранной ноты.

### 4.3 Chord lane

- блоки аккордов имеют `symbol`, `start`, `duration`;
- блок можно выбрать, двигать и растягивать;
- допускаются разрывы и перекрытия, но UI показывает предупреждение;
- аккорд определяется для ноты по её `start`; позднее появится режим оценки
  всей длительности ноты.

### 4.4 Цветовые режимы

`Roles` — основной режим:

| Роль | Смысл |
|---|---|
| Root | корень текущего аккорда |
| Guide | терция и септима |
| Chord | остальные тоны аккорда |
| Tension | доступные 9/11/13 и альтерации |
| Scale | ладовые, но не аккордовые тоны |
| Outside | хроматические тоны вне выбранного контекста |

`Degrees` сохраняет постоянный цвет ступеней I–VII, как в лаборатории.

Цвет не является запретом. Outside note остаётся доступной и только получает
диагностическую окраску.

## 5. Работа с гармонией

Источники гармонии:

1. JSON-проект из ReferenceCompare — приоритет MVP;
2. ручной ввод и редактирование chord lane;
3. в будущем — `Learn from Chord Track`.

Поддерживаются:

- трезвучия и септаккорды;
- extensions и alterations;
- slash chords и обращения;
- borrowed/secondary chords;
- один глобальный лад плюс локальный контекст каждого аккорда.

Музыкальная модель должна использовать существующий движок
`ReferenceCompare/app/composer_lab.py` как эталон поведения, но не копировать
бизнес-логику в несколько несовместимых реализаций без контрактных тестов.

## 6. Обмен с Ableton Live

### Pull Selected Clip

- источником служит `Song.View.detail_clip`;
- операция разрешена только для MIDI-клипа;
- читаются все ноты и основные clip metadata;
- note IDs сохраняются для безопасного последующего Commit;
- Pull никогда не изменяет Live Set.

### Commit

- проверяется идентичность связанного clip locator;
- перед записью сравнивается revision/fingerprint;
- если клип изменился снаружи, пользователь выбирает Reload или Force Commit;
- операция должна быть одной отменяемой пользователем транзакцией Live, если
  API это позволяет;
- ни одна невалидная нота не отправляется в Live.

### Commit as New

- создаёт новый MIDI-клип рядом с исходным или в выбранном slot;
- исходный клип не изменяется;
- имя получает суффикс варианта.

### Drag MIDI

- создаётся SMF Type 1 или Type 0 с PPQ не ниже 480;
- временный файл не удаляется в completion callback drag operation: Live может
  прочитать его позже;
- старые временные файлы очищаются при следующем запуске или по TTL;
- в файл записываются tempo, time signature и clip name, где это уместно.

## 7. MIDI routing template

```text
HC — HARMONY CANVAS
    ↓ MIDI From
HC — BRIDGE (Monitor In)
    ↓ MIDI To
HC — INSTRUMENT / RECORD
```

Template также содержит служебную дорожку Harmony Guide и сохранённые настройки
мониторинга, чтобы пользователь не собирал routing вручную.

## 8. Надёжность и безопасность данных

- Pull по умолчанию read-only;
- Commit недоступен без сохранённого locator исходного клипа;
- любое внешнее изменение клипа обнаруживается fingerprint-проверкой;
- сетевой разрыв не очищает редактор;
- состояние проекта сохраняется внутри Live Set и дублируется локальным
  recovery snapshot;
- все сообщения имеют `protocol_version` и `request_id`;
- неизвестные поля игнорируются, неизвестные версии отклоняются явно.

## 9. Производительность MVP

- 2 000 нот: первичная отрисовка до 200 мс на целевой машине;
- drag/resize: визуальное обновление около 60 FPS;
- Pull 2 000 нот: до 1 секунды;
- гармоническая переоценка при изменении одного аккорда: до 50 мс;
- UI не выполняет файловые или сетевые операции в audio thread.

## 10. Критерии готовности MVP

MVP принят, когда выполняется сквозной тест:

1. открыть template в Live 12;
2. создать MIDI-клип с минимум 20 нотами;
3. Pull переносит его без изменения pitch/time/duration/velocity;
4. прогрессия из четырёх аккордов отображается на общей оси;
5. строки и ноты меняют гармоническую роль во времени;
6. пользователь добавляет и перемещает ноты;
7. playback слышен через receiver instrument;
8. Commit обновляет исходный клип;
9. Undo в Live возвращает предыдущее состояние;
10. drag-out создаёт новый корректный MIDI-клип в Live;
11. повторное открытие Live Set восстанавливает состояние Harmony Canvas.

## 11. Отложено после MVP

- генераторы melody skeleton/approach notes/motif adaptation;
- автоматический voice leading;
- оценка сильных и слабых долей;
- несколько мелодических голосов;
- MPE, probability и velocity deviation editing;
- chord-track recognition;
- macOS и другие DAW;
